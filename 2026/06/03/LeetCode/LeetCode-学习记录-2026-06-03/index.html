🔍 关键角色详解

unordered_map<int, int> in_map — 中序值→下标映射

in_left / in_right — 当前子树在中序数组中的范围

两者协作流程（以 preorder=[3,9,20,15,7] 为例）：

  1. pre_idx=0 → 取 3，查 in_map[3]=1 → 左范围 [0,0]，右范围 [2,4]
  2. pre_idx=1 → 取 9，查 in_map[9]=0 → 左穷，右穷 → 叶子 ✅
  3. pre_idx=2 → 取 20，查 in_map[20]=3 → 左 [2,2]，右 [4,4]
  4. …

类比：in_map 是索引页（查词在第几页），in_left/in_right 是当前阅读的页码范围（超出范围不看）。

  • pre_idx 必须传引用（int &），所有递归层共享全局递增
  • 必须先建左子树再建右子树（前序顺序决定的）
  • 空判断用 in_left > in_right（中序区间为空），不用检查 preorder 空
  • 一句话记：”前序定根 → 中序分左右 → pre_idx 全局引用 → 先左后右”

[Medium] Lowest Common Ancestor of a Binary Tree (236)

🎯 核心思路

后序遍历，自底向上。遇到 p 或 q 就返回，左右子树各返回一个结果，合并判断：

  • 左空→返回右，右空→返回左，都非空→返回 root

📚 对比原解法

⚠️ 易错点 & 反思

  • 终止条件 3 合 1：!root || root==p || root==q，遇目标即返回
  • 后序保证从底向上：左右结果都回来后才做判断
  • 口诀：”搜到目标就返回，只在左右各找到一个时才是 LCA”
  • 与 235（BST 版 LCA）的区别：BST 版用值大小三向比较可以剪枝单路递归；普通树只能全量搜索

🔍 三个例子走一遍

例1：p=5, q=4（p 是 q 的祖先）

例2：p=6, q=4（同一子树不同分支）

例3：p=7, q=8（根的两侧）

核心观察：后序保证了从底向上的返回顺序。一旦某节点左右各找到一个目标，它自己就是 LCA，不再需要往上找了。
