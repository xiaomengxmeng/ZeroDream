📝 2026-06-11 学习思考过程 — 动态规划 Day 4：双序列 DP

Day 3 完成了零钱兑换 + LIS ✅，今天进入双序列 DP：LCS（最长公共子序列）

[Medium] 最长公共子序列 (1143)

🤔 我的原始思路

  • dp[i][j] 表示两个字符串以对应位置结尾的最长 LCS 的长度
  • text1[i] == text2[j] 时这个字符一定在 LCS 中
  • dp 表需要处理边界，建 dp(m+1, vector(n+1, 0))，下标从 1 开始

🔍 我的思考过程

  1. 状态定义：dp[i][j] = text1[0..i-1] 和 text2[0..j-1] 的 LCS 长度（用 size+1 处理边界）
  2. 相等时：text1[i-1] == text2[j-1] → dp[i][j] = dp[i-1][j-1] + 1
  3. 不相等时：dp[i][j] = max(dp[i-1][j], dp[i][j-1]) — 跳过 text1[i-1] 或 text2[j-1]
  4. 边界：dp[0][] = dp[][0] = 0，空字符串没有公共子序列
  5. 初始化：用 vector<vector<int>> dp(m+1, vector<int>(n+1, 0))

💡 对比收获

  • LIS → LCS：从一维”以 i 结尾”扩展到二维”前 i 和前 j”，两个不同的”结尾”视角
  • 新的转移模式：相等时左上+1，不等时取左/上最大值 — 不同于 LIS 的遍历所有 j
  • size+1 技巧：统一处理边界，避免 i-1/j-1 越界判断
  • 与 62/63 二维路径对比：62 只能向右/下走，LCS 可以”跳过”字符，转移不同
  • 空间优化：二维 DP 只依赖三格（上、左、左上），可优化为一维滚动数组，空间 O(n)

[Medium] 最长回文子序列 (516)

🤔 我的原始思路

回文子序列
直接反转后LCS

🔍 我的思考过程

  1. 关键洞察：回文正反读一样 → LPS(s) = LCS(s, reverse(s))
  2. 直接复用刚写好的 LCS 代码，反转 s 后调用
  3. 一行代码解决核心逻辑

💡 对比收获

  • 回文 ↔ LCS 的等价转换：这是 DP 中”问题转化”的经典案例 — 不写新 DP，而是把新问题映射到已解决的问题
  • 区间 DP 备选：也可以直接在原串上 dp[i][j] 表示 s[i..j] 的 LPS
  • 两种方法核心转移一样：相等 +2 / 不等取 max
  • 复用的力量：1143 写完，516 几乎零成本解决

[Medium] 最长回文子串 (5)

🤔 我的原始思路

子串必须连续
以每个字符开始扩散找到所有回文子串
dp[i][j]表示s[i,j]为回文子串

🔍 我的思考过程

  1. 先写中心扩散：expand(l, r) 向两边扩散，while 结束后回文范围是 [left+1, right-1]
  2. 踩坑：expand 返回 {left, right} 导致 substr 取错，修正为 {left+1, right-1}
  3. 对比写法：更简洁的写法是 expand 直接返回长度 (r - l - 1)，用 start + maxLen 记录
  4. 区间 DP 留到明天：按长度从小到大遍历，dp[i][j] 为 bool

💡 对比收获

  • 中心扩散法：O(n²) 时间，O(1) 空间，面试常用
  • substring vs subsequence：516 可跳字符→LCS 转化，5 必须连续→中心扩散
  • 区间 DP 遍历顺序：按长度从小到大（和 LCS 按下标顺序不同！）
  • 三种回文 DP 排布：
    • 516 LPS：dp[i][j] = 长度，可跳字符
    • 5 LPS：dp[i][j] = bool，连续
    • 647 回文计数：bool dp + count

📚 2026-06-11 学习总结 — 动态规划 Day 4：双序列 DP

从 LIS 到 LCS：一维”以 i 结尾” → 二维”前 i 和前 j”，双序列 DP 入门

[Medium] 最长公共子序列 (1143)

🎯 核心思路

二维 DP：dp[i][j] 表示 text1[0..i-1] 和 text2[0..j-1] 的 LCS 长度。关键转移：

  • 末尾字符相等 → 接上这个字符：dp[i][j] = dp[i-1][j-1] + 1
  • 末尾不等 → 取跳过其中一个的最大值：dp[i][j] = max(dp[i-1][j], dp[i][j-1])

🔑 关键代码片段

📚 学到的知识点

  • 双序列 DP 框架：两个序列/字符串的比较问题，常用 dp[i][j] 表示”前 i 和前 j”
  • 三种转移来源：左上（匹配）、上（跳过 text1）、左（跳过 text2）— 二维 DP 的经典三角依赖
  • size+1 技巧：dp(m+1)×(n+1) 让边界自动为 0，下标 1-based，简化代码
  • LIS → LCS：一维”以 i 结尾”扩展到二维”前 i 和前 j”，子序列 DP 的两种视角
  • 滚动数组优化：dp[i][j] 只依赖 dp[i-1][j-1]、dp[i-1][j]、dp[i][j-1]，可优化为 O(n) 空间

⚠️ 易错点 & 反思

  • ⚠️ 下标对应：dp[i][j] 对应 text1[i-1]、text2[j-1]，不是 text1[i]、text2[j]
  • ⚠️ 不要写成 dp(i, j) 从 0 开始，否则 i-1/j-1 越界需要额外 if 判断
  • ✨ 和 62/63 对比：62 的二维 DP 路径只能向右/下，LCS 可以”跳过”字符，机动性更大
  • ✨ 和 LIS 对比：LIS 需要遍历所有 j < i，LCS 只需看 i-1/j-1 这一步——牺牲一维换来了 O(1) 转移

[Medium] 最长回文子序列 (516)

🎯 核心思路

回文正反读一样 → LPS(s) = LCS(s, reverse(s))。将原问题转化为已解决的 LCS 问题，复用 1143 的代码即可。一行核心逻辑：return longestCommonSubsequence(s, string(s.rbegin(), s.rend()))。

🔑 关键代码片段

📚 学到的知识点

  • 问题转化：将回文问题映射为 LCS，不需设计新 DP
  • **LPS(s) = LCS(s, reverse(s))**：回文的对称性决定反转后不变
  • 区间 DP 替代方案：dp[i][j] 表示 s[i..j] 的 LPS，s[i]=s[j] 时 +2 向两边扩展
  • 复用思维：写完 LCS 后 LPS 几乎零成本解决，体现了模块化复用的价值

⚠️ 易错点 & 反思

  • ⚠️ 反转字符串用 s.rbegin(), s.rend() 而非额外循环
  • ⚠️ 注意是 subsequence 不是 substring — 可以跳字符
  • ✨ 连续版见 5. Longest Palindromic Substring（不能跳字符，解法不同）
  • ✨ 对比区间 DP：LCS 法写法简单，区间 DP 空间可优化

🔗 关联题目

    1. Longest Palindromic Substring（连续回文子串）
    1. Palindromic Substrings（回文子串计数）
    1. Delete Operation for Two Strings（m + n - 2*LCS）

[Medium] 最长回文子串 (5)

🎯 核心思路

中心扩散法：每个位置作为中心向两边扩展，检查回文。区分奇数长度（单字符中心）和偶数长度（双字符中心）。用 expand 函数返回回文长度，start + maxLen 记录全局最长。

🔑 关键代码片段

📚 学到的知识点

  • 中心扩散法：O(n²) 时间 O(1) 空间，面试常考，实现简单
  • 奇偶统一：奇数中心是 i,i，偶数中心是 i,i+1，用 max 取较大者
  • start 计算：i - (len-1)/2 从中心回推到起始位置

⚠️ 易错点 & 反思

  • ⚠️ expand 返回 r - l - 1 而不是 r - l + 1（while 多走了一步）
  • ⚠️ 奇偶分开处理，不要只考虑奇数中心
  • ✨ 区间 DP 版留到明天练习：按长度从小到大遍历，dp[i][j] = bool
  • ✨ substring 和 subsequence 的解法完全不同（对比 516）

    1. Palindromic Substrings（计数版，也是中心扩散）
    1. Shortest Palindrome（中心扩散 + KMP）
