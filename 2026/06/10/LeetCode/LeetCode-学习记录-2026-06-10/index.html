📝 2026-06-10 学习思考过程 — 动态规划 Day 3：最值型 DP 进阶

Day 2 完成了二维路径 + 完全平方数 ✅，今天进入零钱兑换（279 的同类扩展）

[Medium] 零钱兑换 (322)

🤔 我的原始思路

  • 每个 amount 尝试减去 coins[i]
  • 贪心：sort(coins)，先取最大的，大于 amount 就换小的
  • 初始化为 INT_MAX/2
  • dp[amount] == INT_MAX/2 return -1; else return dp[amount]

🔍 我的思考过程

  1. 贪心不可行：coins=[1,3,4], amount=6 时贪心 4+1+1=3 不如 3+3=2
  2. 回到 279 的框架：dp[i] = min(dp[i - coin] + 1) over all coins
  3. 初始化问题：coins 可能不含 1，用 INT_MAX/2 作为不可达标记，避免溢出
  4. 跳过条件：dp[i - coin] 不可达时跳过 min 比较
  5. 循环边界：踩坑 → i <= amount 而不是 <，否则 dp[amount] 永不计算

💡 对比收获

  • 322 和 279 是同一类最值 DP，面额来源不同（平方数 vs 数组）
  • 贪心在通用面额下可能失败，DP 保证最优
  • 不可达标记 + 跳过条件是最值型 DP 处理”无解”的标准技巧
  • 统一框架：70（面额1/2）、279（平方数）、322（数组）→ 同一模板

[Medium] 最长递增子序列 (300)

🤔 我的原始思路

  • dp[i] 表示以 nums[i] 结尾的 LIS 长度
  • “转移方程一维只需要一个，不需要所有”（初稿，后来修正）
  • 维护全局最大值 ans

🔍 我的思考过程

  1. 状态定义：dp[i] = 以 nums[i] 结尾的最长递增子序列长度
  2. 最初的误区：以为转移只需要看前一个元素，实际需要遍历所有 j < i
  3. 修正：dp[i] = max over j < i (dp[j] + 1) 当 nums[j] < nums[i] — 和 322 遍历硬币本质一致
  4. 全局最大值：答案是所有 dp[i] 的 max，不是 dp[n-1]（LIS 不一定以最后一个元素结尾）
  5. 边界细节：ans 初始化为 1 而不是 0（n=1 时不会进循环，但答案应为 1）

💡 对比收获

  • 新的状态定义模式：”以 i 结尾” vs “凑成目标 i” — 两种不同的 DP 视角
  • 遍历对象不同：322 往前减 coin，300 往前遍历 j — 但都是”外层目标 + 内层候选”的结构
  • dp 初始化为 1：每个元素自身至少构成长度 1 的子序列
  • ans 初始化：n≥1 时至少返回 1，n=0 不会出现（题目约束）
  • O(n log n) 优化：贪心 + 二分（tails 数组）替代 O(n²)
    • lower_bound 找第一个 ≥ num 的位置（就是 35 题 Search Insert Position）
    • 替换成更小的值，贪心让后面的数更容易接上
    • tails 不是 LIS 本身，只保证长度正确

📚 2026-06-10 学习总结 — 动态规划 Day 3：最值型 DP 进阶 + 子序列 DP

从完全平方数到零钱兑换：统一的最值 DP 框架 + 以 i 结尾的子序列 DP 入门

[Medium] 零钱兑换 (322)

🎯 核心思路

一维最值型 DP：dp[i] 表示凑成金额 i 所需的最少硬币数。对每个金额 i，遍历所有硬币面额 coin，取最小值。与 279 完全平方数是同一框架，但面额来自任意数组，且需处理不可达情况。

🔑 关键代码片段

📚 学到的知识点

  • 统一 DP 框架：dp[i] = min over 面额 (dp[i - 面额] + 1)，70/279/322 本质相同
  • 不可达标记：INT_MAX/2 作为占位符，避免加 1 溢出，且能参与 min 比较
  • 遍历硬币：用 range-based for 代替下标循环，更安全
  • 条件跳过：dp[i - coin] 不可达时跳过，避免无效计算

⚠️ 易错点 & 反思

  • ⚠️ 循环边界 i <= amount 而不是 <，否则 dp[amount] 永远算不出来
  • ⚠️ INT_MAX 直接 +1 会溢出，用 INT_MAX/2 安全
  • ✨ 与 279 的关系：同一类最值问题，区别在于面额的来源方式

🔗 关联题目

    1. Climbing Stairs（同框架：求和型，固定面额 1/2）
    1. Perfect Squares（同类：最值型 DP，平方数面额）
    1. Coin Change II（同类：求方案数，变求和）

[Medium] 最长递增子序列 (300)

🎯 核心思路

以 nums[i] 结尾的 LIS 长度：dp[i] = max over j<i (dp[j] + 1) if nums[j] < nums[i]。每个 i 往前遍历所有 j 取最大值，最后对所有 dp[i] 取全局最大。子序列 DP 的经典入门题。

**优化 O(n log n)**：贪心 + 二分。维护 tails 数组，tails[k] 表示长度为 k+1 的递增子序列的最小结尾。对每个 num，用 lower_bound 找第一个 ≥ num 的位置替换或追加，最终 tails.size() 即为 LIS 长度。

🔑 关键代码片段

📚 学到的知识点

  • “以 i 结尾”的状态定义：不同于之前”凑成目标 i”的 DP，LIS 的 dp[i] 是”以 nums[i] 结尾”的，全局答案需要取 max
  • 往前遍历所有 j：和 322 遍历所有硬币是同一模式，但前驱来自数组位置而非减法
  • dp 全初始化为 1：每个元素自身至少长度为 1 的子序列
  • lower_bound 二分：在有序的 tails 数组中找第一个 ≥ num 的位置替换，保证 tails 始终递增
  • 贪心思想：替换成更小的结尾值，让后续元素更容易接上

⚠️ 易错点 & 反思

  • ⚠️ ans 初始化为 1 而非 0：n=1 时不会进入循环，ans=0 会错误返回 0
  • 答案在 dp 数组的 max 而非 dp[n-1]：LIS 不一定以最后一个元素结尾
  • 和 322 对比：外层遍历目标（amount/下标），内层遍历候选（硬币/j）取最值
  • O(n log n) 的 tails 数组不是 LIS 本身，只保证长度正确
  • lower_bound 对应之前做的 35. Search Insert Position — 找第一个 ≥ target 的位置

🔗 关联题目

    1. Longest Continuous Increasing Subsequence（连续版，O(n)）
    1. Russian Doll Envelopes（二维 LIS）
