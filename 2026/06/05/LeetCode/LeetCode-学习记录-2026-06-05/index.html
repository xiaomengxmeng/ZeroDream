[Medium] Combination Sum (39)

🎯 核心思路

回溯 + 排序剪枝：用 start 索引防重复组合，可重复选取时传 i（而非 i+1），排序后 sum + candidates[i] > target → break 剪枝。

🔑 关键代码片段

📚 对比原解法

一致 ✅ — 排序剪枝思路完全正确，传 i 实现可重复选取

⚠️ 易错点 & 反思

  • 排序是剪枝的前提：只有升序排列才能保证 candidates[i+1] >= candidates[i]，遇到超 target 的直接 break
  • 可重复选取传 i，不可重复传 i+1（这是 39 和 40 的核心区别）
  • sum 传值（无需回溯），path 传引用（需 pop_back）
  • 可以用 last 递减版：dfs(path, last - candidates[i], candidates, i, res) — 本质等效
