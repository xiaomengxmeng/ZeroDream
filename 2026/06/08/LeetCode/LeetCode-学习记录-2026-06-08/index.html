📝 2026-06-08 学习思考过程 — 动态规划入门 Day 1

[Easy] 爬楼梯 (70)

🤔 我的原始思路

  1. 小规模枚举：n=1 → ?， n=2 → ?， n=3 → ?
  • n =1 一种 n =2 两种 n=3 3种
  1. 找规律：到达第 i 阶的最后一步，可以从哪两阶跨上来？
  • 到达i阶的最后一步只能从 i-1 好i-2 跨上来
  1. 递推公式：dp[i] = ?
  • dp[i] = dp[i-1] +dp[i-2];

🔍 我的思考过程

  • 从小规模枚举找规律：n=1→1, n=2→2, n=3→3，发现是斐波那契数列
  • 最后一步思维：到第 i 阶只能从 i-1 或 i-2 跨上来，所以 dp[i] = dp[i-1] + dp[i-2]
  • dp 数组从 1 开始，n≤2 直接返回

💡 对比收获

  • DP 三要素：状态定义 dp[i]、转移方程、初始条件
  • 这是所有 DP 题的最基础模型 — 斐波那契型 DP
  • 空间可优化为滚动变量 O(1)，但数组更直观展示 DP 结构

[Easy] 使用最小花费爬楼梯 (746)

🤔 我的原始思路

dp[i] 定义：爬到第 i 阶的cost

（超过数组末尾）

res=min(dp[n],dp[n-1])dp[0]=dp[1]=0;

dp[i]=min(dp[i-1]+cost[i-1],dp[i-2]+cost[i-2])

🔍 我的思考过程

  • dp[i] 定义：爬到第 i 阶的最小花费（i 从 0 到 n，n=cost.size()）
  • 初始条件：dp[0]=dp[1]=0（第 0/1 阶不需要花钱就能站上去）
  • 转移方程：dp[i] = min(dp[i-1]+cost[i-1], dp[i-2]+cost[i-2])
  • 最终答案：dp[n]（超过数组末尾）
  • 注意：循环从 i=2 开始，否则 i-1/i-2 会越界

💡 对比收获

  • 与 70 题的区别：从”方法数求和”变成”最小花费取 min”
  • 70 是 dp[i]=dp[i-1]+dp[i-2]，746 是 dp[i]=min(dp[i-1]+cost[i-1], dp[i-2]+cost[i-2])
  • 70 求总数用 +，746 求最值用 min/min
  • 初始化的含义不同：746 可以免费站在起点，所以 dp[0]=dp[1]=0

[Medium] 打家劫舍 (198)

🤔 我的原始思路

第i家偷或者不偷

dp[i] 前 i+1 家（0~i）中能偷到的最高金额

dp[i] =max(dp[i-2]+nums[i],dp[i-1]);

dp[0]=nums[0];

dp[1]=max(nums[0],nums[1])

ans=dp[n]

🔍 我的思考过程

  • 与 746 的核心区别：746 每步必选（取 min），198 可选偷或不偷（取 max）
  • dp[i] 表示前 i+1 家（0~i）能偷到的最高金额
  • 偷第 i 家 → dp[i-2] + nums[i]；不偷 → dp[i-1] → 两者取 max
  • 初始化：dp[0]=nums[0], dp[1]=max(nums[0], nums[1])
  • 注意边界：n==0 返回 0, n==1 返回 nums[0]

💡 对比收获

  • 递推三连的变化轨迹：70（+求和）→ 746（min 求最值）→ 198（max 求最值 + 可选跳过）
  • 70 和 746 都是”必走”（每步都要从 i-1 或 i-2 过来），198 是”可选跳过”
  • 偷不偷 = 选不选 = max 的语义
  • 198 的 bug 排查经验：= vs ==、n==1 边界、dp[n-1] 而非 dp[n] — 都是经典坑

[Easy] 爬楼梯 (70)

🎯 核心思路

斐波那契型 DP：状态定义 dp[i] 为爬到第 i 阶的方法数，转移方程 dp[i] = dp[i-1] + dp[i-2]（最后一步从 i-1 或 i-2 跨上来）

🔑 关键代码片段

📚 学到的知识点

  • DP 三要素：状态定义（dp[i]）、转移方程（dp[i-1]+dp[i-2]）、初始条件（dp[1]=1, dp[2]=2）
  • 最后一步思维：只关注”最后一步从哪来”，无需思考中间过程
  • 空间优化：可用滚动变量（a, b, c）将 O(n) → O(1)

⚠️ 易错点 & 反思

  • dp 数组大小应为 n+1，索引从 1 开始
  • n≤2 的情况要提前处理，否则 dp[2] 赋值可能越界
  • 与斐波那契数列的区别：dp[2]=2 而非 1

🔗 关联题目

    1. Fibonacci Number（同类型基础）
    1. Min Cost Climbing Stairs（加 cost 变体）

[Easy] 使用最小花费爬楼梯 (746)

🎯 核心思路

最小花费 DP：dp[i] 表示爬到第 i 阶的最小花费，从 i-1 或 i-2 跨上来取较小值。可在第 0/1 阶免费开始。

🔑 关键代码片段

📚 学到的知识点

  • DAM 型 DP：从求和（+）到求最值（min/min）的转变
  • 初始化差异：起点免费，dp[0]=dp[1]=0
  • 顶部位置：”超过数组末尾” → dp[n] 即答案
  • 与 70 题的核心区别：代价 + 求最值 vs 方法数求和

⚠️ 易错点 & 反思

  • ❌ 循环从 i=0 开始 → i-1/i-2 负索引越界，需从 i=2 开始
  • ❌ 答案误写 min(dp[n], dp[n-1]) → dp[n] 已经是超过末尾的值
  • 与 70 题的区别梳理：70（求和） vs 746（取 min）

🔗 关联题目

    1. Climbing Stairs（同框架，求和方法数）
    1. House Robber（不能相邻 + 取 max）

[Medium] 打家劫舍 (198)

🎯 核心思路

“选或不选”型 DP：dp[i] 表示前 i+1 家（0~i）能偷到的最高金额。偷第 i 家 → dp[i-2] + nums[i]；不偷 → dp[i-1] → 取 max。

🔑 关键代码片段

📚 学到的知识点

  • 递推三连的演变路径：70（+求和）→ 746（min 求最值）→ 198（max 求最值 + 可选跳过）
  • “选或不选”模型：可跳过元素时，dp[i] = max(选, 不选)
  • 三题框架统一：都是线性 DP，状态依赖前两个位置

⚠️ 易错点 & 反思

  • ❌ if (n = 0) → 赋值不是比较，必须用 ==
  • ❌ n == 1 时 dp[1] = max(nums[0], nums[1]) 越界 — 要提前返回
  • ❌ return dp[n] → dp 大小为 n，应返回 dp[n-1]
  • 空间可优化为滚动变量 O(1)（存 prev2, prev1, curr 三个变量）

🔗 关联题目

    1. Climbing Stairs（同框架，求和方法数）
    1. Min Cost Climbing Stairs（同框架，求最小花费）
    1. House Robber II（环状版本进阶）
