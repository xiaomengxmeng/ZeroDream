[Medium] Merge Intervals (56)

🤔 我的原始思路

  1. [a,b] 和 [c,d] 中 假设a< c ，则b>=c时重叠
  2. 无序先按每一列的第一个元素排序
  3. 使用双层数组来存储 新区间的起点 是 [a,max(b,d)]
  4. 直接修改结果数组

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：合并所有重叠区间
  2. 重叠定义：[a,b] 和 [c,d]，若 a < c 且 b >= c，则重叠
  3. 关键洞察：无序区间难以处理，需要先排序

Step 2: 设计方案

  • 排序策略：按区间起点排序
  • 合并逻辑：
    • 维护结果数组，初始放入第一个区间
    • 遍历后续区间，与结果数组最后一个区间比较
    • 若重叠：合并（终点取 max）
    • 若不重叠：添加新区间
  • 合并公式：[a, max(b, d)]

Step 3: 代码实现

Step 4: 代码分析

  • ✅ 排序：让区间有序，便于顺序处理
  • ✅ **res.back()**：获取结果数组最后一个区间
  • ✅ 原地更新终点：合并时直接修改最后一个区间的终点
  • ✅ 边界处理：空数组直接返回

测试用例验证：

Step 5: 最终理解

  • 区间问题套路：排序 → 遍历 → 合并/添加
  • 排序的作用：让重叠区间相邻，便于顺序处理
  • res.back() 技巧：始终维护”当前合并区间”

💡 对比收获

原始想法 vs 最终实现：

  • ✅ 思路正确：排序 + 合并
  • ✅ 合并公式正确：[a, max(b,d)]
  • ✅ 代码简洁高效

新学到的知识点：

  1. 区间问题通用套路：排序 → 遍历 → 合并
  2. 二维数组排序：sort() 默认按第一个元素排序
  3. res.back() 技巧：获取最后一个元素，便于比较和修改
  4. 端点相连也算重叠：[1,4] 和 [4,5] 可合并

[Easy] Add Binary (67)

🤔 我的原始思路

二进制无符号数加法 从两个字符串的最后相加 上一位进位1
0+0 =0 0+1 =1 1+1 =10 1+1+1 =11(处理进位)

一个字符串长度不足时 后只需处理进位了，直接正向拼接最后反转 当前位计算完后更新carry和结果
字符转数字 -‘0’

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：两个二进制字符串相加
  2. 与 Plus One 的相似之处：从后往前、处理进位
  3. 不同之处：两个数相加，需要处理长度不同的情况

Step 2: 设计方案

  • 遍历方向：从后往前（从低位到高位）
  • 循环条件：i>=0 || j>=0 || carry — 三个条件都要满足
  • 当前位计算：sum % 2
  • 进位计算：sum / 2
  • 结果构建：正向拼接，最后反转

Step 3: 代码实现

Step 4: 代码分析

  • ✅ 循环条件完美：处理了长度不同和最后进位的情况
  • ✅ sum / 2 和 sum % 2 巧妙计算进位和当前位
  • ✅ to_string() 简洁地将数字转为字符

测试用例验证：

💡 对比收获

与 Plus One 的对比：

维度 Plus One (66) Add Binary (67)
操作 只加1 两个数相加
数据结构 数组 字符串
进位处理 相同 相同
遍历方向 从后往前 从后往前

新学到的知识点：

  1. 通用进位模式：carry = sum / base，current = sum % base
  2. 多操作数相加：循环条件要包含所有操作数和进位
  3. 字符串反转技巧：正向拼接后反转，比反向插入更高效
  4. to_string() 函数：快速将数字转为字符串

[Medium] Find All Anagrams in a String (438)

🤔 我的原始思路

先得到p的长度，在编写函数判断是否是异位词，然后逐个判断s的子串是否是
暴力时间复杂度 O(n*m)? 优化 相邻子串就一个字母不同 如果 子串的开头字母和子串的下一个字母相同，那下一个子串和当前的就是异位词 就可以减少和p的判断次数

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：在 s 中找所有 p 的异位词子串
  2. 与 Valid Anagram 的关系：需要判断多个子串
  3. 暴力解法：O(n * m)，每次重新计算频率

Step 2: 设计方案（滑动窗口）

  • 窗口大小：固定为 p.size()
  • 核心优化：相邻窗口只差一个字符，增量更新
  • matched 变量：记录匹配的字符数
  • 更新规则：
    • 进入字符：count[c]--，若变成 0 则 matched++，若变成 -1 则 matched–
    • 离开字符：count[c]++，若变成 0 则 matched++，若变成 1 则 matched–

Step 3: 代码实现

Step 4: Bug 修复

  • ❌ 最初用遍历方式计算 matched，失去滑动窗口优势
  • ❌ 进入/离开字符时检查了错误的字符
  • ❌ 索引计算错误：起始索引应该是 i - p.size() + 1
  • ✅ 修复：增量更新 matched，O(1) 时间

💡 对比收获

与 Valid Anagram 的对比：

维度 Valid Anagram (242) Find Anagrams (438)
操作 判断两个字符串 找所有子串
方法 频率统计 滑动窗口 + 频率统计
复杂度 O(n) O(n)

新学到的知识点：

  1. 滑动窗口模板：固定窗口大小 + 增量更新
  2. matched 增量更新：O(1) 判断是否匹配
  3. count 变化的含义：0→匹配，±1→不匹配
  4. 窗口索引：起始 = i - window_size + 1

[Medium] Merge Intervals (56)

🎯 核心思路

先按区间起点排序，然后遍历合并。维护结果数组，每次将当前区间与结果数组最后一个区间比较：若重叠则合并（更新终点为 max），否则添加新区间。时间复杂度 O(n log n)，空间复杂度 O(n)。

🔑 关键代码片段

📚 学到的知识点

  • 区间问题通用套路：排序 → 遍历 → 合并/添加
  • 二维数组排序：sort() 默认按第一个元素排序
  • res.back() 技巧：获取最后一个元素，便于比较和修改
  • 重叠判断：end >= next_start 表示重叠（端点相连也算重叠）
  • 合并公式：[start, max(end1, end2)]

⚠️ 易错点 & 反思

  • ❌ 忘记排序直接遍历，导致无法正确合并
  • ❌ 重叠判断条件写错：应该是 >= 而不是 >
  • 💡 排序的作用：让重叠区间相邻，便于顺序处理
  • 💡 端点相连也算重叠：[1,4] 和 [4,5] 可合并为 [1,5]
