[Easy] Valid Parentheses （20）

🤔 我的原始思路

  • 经典的括号匹配题目
  • 使用栈结构 来判断括号是否有效
  • ( [ { 入栈 遇到 ) ] } 尝试出栈 如果栈顶元素与当前的不匹配 return false
  • 直到遍历完串

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：左括号必须用同类型的右括号闭合，且顺序正确
  2. 数据结构选择：栈（LIFO特性完美匹配嵌套结构）
  3. 基本思路：左括号入栈，右括号出栈匹配

Step 2: 初版代码实现

  • 左括号：直接入栈
  • 右括号：检查栈顶是否匹配，匹配则出栈
  • 最后检查栈是否为空

Step 3: 发现Bug

  • ❌ 原始代码：if (stack.top() == '{' || stack.empty())
  • 问题：当栈为空时调用 stack.top() 会导致未定义行为
  • 💡 修复：必须先检查 stack.empty()，再访问 stack.top()

Step 4: 进一步优化

  • 发现逻辑问题：stack.empty() 检查位置不对
  • 原问题：每次循环都检查（包括左括号刚入栈后）
  • 正确做法：只在处理右括号时检查栈是否为空
  • 理由：只有右括号需要找到对应的左括号，此时才可能遇到栈为空的情况

Step 5: 最终理解

  • 栈的本质：保存”期待被匹配的左括号”
  • 栈顶永远是最近一个未匹配的左括号
  • 边界情况：
    • 输入只有右括号 → 立即返回false
    • 遍历结束栈不为空 → 有未匹配的左括号
    • 栈空时访问top → 必须先检查

💡 对比收获

原始想法 vs 最终理解：

  • ✅ 思路正确：使用栈进行括号匹配
  • ✅ 发现并修复了关键bug：空栈访问问题
  • ✅ 优化了逻辑流程：空检查位置调整
  • ✅ 理解了栈的本质：LIFO特性与嵌套结构的完美匹配

新学到的知识点：

  1. 栈的基本操作原则：访问前必须确保不为空
  2. 括号匹配的本质：配对/匹配模式
  3. 边界情况的系统化处理方法
  4. 代码优化的思路：从能运行到优雅高效

[Easy] Min Stack（155）

🤔 我的原始思路

  • 常数时间内检索到最小元素的栈
  • 常规栈的 getMin() 操作时间复杂度是多少？O(n)
  • 维护最小值，每次入栈时判断是否需要更新min
  • 辅助栈方案
  • 主栈是正常栈，辅助栈初始为空
  • 元素入栈时：如果辅助栈为空或者辅助栈顶大于等于元素时，入栈
  • 主栈出栈时：如果辅助栈顶元素与主栈弹出值相同也出栈
  • 时间复杂度O(1)空间复杂度O(n)

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：设计一个支持 push、pop、top、getMin 操作的栈
  2. 特殊要求：getMin() 必须在 O(1) 时间内完成
  3. 常规栈的问题：getMin() 需要遍历整个栈，时间复杂度 O(n)
  4. 解决方向：使用辅助数据结构来跟踪最小值

Step 2: 设计方案（辅助栈）

  • **主栈 (main_stack)**：存储所有元素，正常进行 push、pop、top 操作
  • **辅助栈 (min_stack)**：存储当前的最小值历史
  • **push(val)**：
    • 主栈正常入栈
    • 如果辅助栈为空 或 val ≤ 辅助栈顶，则 val 也入辅助栈
  • **pop()**：
    • 先保存主栈的栈顶值
    • 主栈弹出
    • 如果保存的值 == 辅助栈顶，则辅助栈也弹出
  • **getMin()**：返回辅助栈顶元素（即当前最小值）

Step 3: 初版代码实现

  • 定义了 MinStack 类
  • 包含 main_stack 和 min_stack 两个成员变量
  • 实现了所有5个方法

Step 4: 发现并修复问题

  • ❌ 成员变量定义位置错误：最初放在构造函数内部（局部变量）
  • ❌ 方法调用错误：isempty() 应该是 empty()
  • ❌ pop() 参数错误：pop(val) 不接受参数
  • ❌ pop() 逻辑错误：先弹出再访问 top() 导致错误
  • 💡 正确做法：
    • 成员变量声明在类级别
    • pop() 不带参数
    • 先保存 top 值，再弹出，最后比较

Step 5: 最终理解

  • 辅助栈的本质：维护一个”最小值的递减序列”
  • **为什么用 >= 而不是 >**：处理重复的最小值情况
    • 例如：push(0), push(-3), push(-3), pop()
    • 如果只用 >，第二次 -3 不会入辅助栈，第一次 pop 后 min 就错了
  • 时间/空间权衡：用 O(n) 的额外空间换取 O(1) 的查询时间

💡 对比收获

原始想法 vs 最终理解：

  • ✅ 思路完全正确：辅助栈是标准解法
  • ✅ 掌握了辅助数据结构的设计思想
  • ✅ 学会了 C++ 类的正确编写方式
  • ✅ 理解了”空间换时间”的具体应用

新学到的知识点：

  1. 辅助栈技术：用于跟踪主栈的某些属性
  2. C++ 类的设计：成员变量的作用域和生命周期
  3. 边界情况处理：重复最小值的处理（>= vs >）
  4. 设计模式：组合多个简单数据结构实现复杂功能

[Easy] Valid Anagram（242）

🤔 我的原始思路

  • 直接统计字符然后判断是否相等就行
  • 是昨天 Group Anagrams 题目的简单版
  • 只有两个字符串，不需要分组

🔍 我的思考过程

Step 1: 问题回顾

  • 这是复习题，基于昨天的 Group Anagrams (49)
  • 昨天用的是排序法，今天尝试用更优的方法

Step 2: 方法选择

  • 排序法：O(n log n) 时间，简单直观
  • 频率统计法：O(n) 时间，更高效 ⭐ 选择这个！
  • 为什么选频率统计法？因为题目只涉及小写字母（26个），可以用固定大小的数组

  • 遍历第一个字符串 s：对应位置计数器 ++（统计出现次数）
  • 遍历第二个字符串 t：对应位置计数器 –（抵消出现次数）
  • 最后检查：如果所有计数器都归零 → 是异位词；否则不是

Step 5: 对比反思

[Easy] Plus One (66)

🤔 我的原始思路

  1. 表示的数字+1，最后一位数加1
  2. 数字大于等于10要进位，数字/10，上一位数+1
  3. 数字为 0 或 999 等特殊情况

🔍 我的思考过程

Step 1: 问题分析

  1. 核心要求：数组表示的数字加1，处理进位
  2. 基本操作：最后一位加1，处理可能的进位
  3. 特殊情况：[9,9,9] → [1,0,0,0]，数组长度增加

Step 2: 设计方案

  • 方法选择：模拟加法（从后往前处理）
  • 基本流程：
    1. 从最后一位开始，加1
    2. 如果该位变成10，则置0，前一位加1
    3. 继续向前处理，直到没有进位
    4. 如果第一位也进位，在数组前面插入1
  • 特殊情况处理：
    • [0] → [1]：正常处理
    • [9,9,9] → [1,0,0,0]：所有位都进位，需要插入新位

Step 3: 代码实现

Step 4: 代码分析

  • ✅ 从后往前遍历：正确处理进位的方向
  • ✅ 提前返回：遇到无进位立即返回，提高效率
  • ✅ 原地修改：不需要额外空间（除了特殊情况）
  • ✅ 正确处理边界：[9,9,9] 这种情况处理完美

Step 5: 最终理解

  • 进位机制：从低位到高位传播，可能传播多位
  • 提前返回优化：一旦无进位就返回，避免不必要的遍历
  • 数组操作技巧：insert() 可以在任意位置插入元素

[Easy] Min Stack (155)

🎯 核心思路

使用辅助栈来跟踪最小值。主栈存储所有元素正常操作，辅助栈存储当前的最小值历史。当新元素小于等于辅助栈顶时同时入辅助栈，弹出时如果主栈弹出的值等于辅助栈顶则也弹出。所有操作的时间复杂度都是 O(1)，空间复杂度 O(n)。

🔑 关键代码片段

📚 学到的知识点

  • 辅助数据结构技术：用额外空间换取查询效率的提升
  • C++ 类的设计：成员变量的作用域和生命周期管理
  • 边界情况处理：重复最小值的处理（使用 >= 而不是 >）
  • “空间换时间”思想：O(n) 空间换取 O(1) 的 getMin() 操作

⚠️ 易错点 & 反思

  • ❌ 成员变量定义在构造函数内部会导致作用域错误（应该是局部变量）
  • ❌ stack.pop() 不接受参数，且无返回值
  • ❌ pop() 时必须先保存 top 值再弹出，否则无法正确比较
  • 💡 关键教训：理解 C++ 类的成员变量与局部变量的区别
  • 💡 **为什么用 >=**：处理重复最小值的情况，确保每次 pop 后辅助栈顶仍是当前最小值

[Easy] Plus One (66)

🎯 核心思路

模拟加法操作，从数组的最后一位开始加1，处理进位。如果某位加1后变成10，则置0并向前进位；如果所有位都进位，则在数组开头插入1。时间复杂度 O(n)，空间复杂度 O(1)（原地修改，特殊情况除外）。

🔑 关键代码片段

📚 学到的知识点

  • 进位处理模式：从低位到高位，遇到10就进位
  • 从后往前遍历技巧：处理进位问题的经典方法
  • 提前返回优化：一旦无进位就返回，减少不必要的遍历
  • 数组操作技巧：insert() 在开头插入元素
  • 大数运算基础：数组表示大数，逐位处理（这是大数加法的简化版）

⚠️ 易错点 & 反思

  • ❌ 进位方向错误：应该从后往前（从低位到高位）处理
  • ❌ 忘记处理 [9,9,9] 这种特殊情况：所有位都进位时需要插入新位
  • ❌ 没有提前返回优化：导致不必要的遍历
  • 💡 关键技巧：原地修改 + 提前返回，既节省空间又提高效率
