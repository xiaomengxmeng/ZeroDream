[Medium] Number of Islands (200)

🎯 核心思路

DFS + 水淹法。遍历网格，遇到 '1' 就计数 +1，然后从该格子开始 DFS 把整个岛屿淹掉（'1'→'0'）。四方向搜索（上、下、左、右），越界或遇到 '0' 就返回。

🔑 关键代码片段

📚 学到的知识点

  • 网格 DFS vs 树 DFS：树有 left/right 两个方向，网格有 4 个方向
  • 水淹法（标记已访问）：直接修改原数组 '1'→'0'，省去 visited 数组
  • 外层双循环扫描：遍历每个格子，遇到陆地就计数并淹掉整个岛屿

⚠️ 易错点 & 反思

  • ❌ 搜索只考虑右和下方 → 会漏掉上方和左方的陆地，必须四方向搜索
  • ❌ 忘记边界检查 i < 0 || i >= m || j < 0 || j >= n
  • ❌ 字符比较用 grid[i][j] == 1 → 应该是 == '1'
  • 💡 水淹法本质是”每次发现新大陆就淹掉它”，相当于每个岛屿只被统计一次

[Medium] Clone Graph (133)

🎯 核心思路

DFS + 哈希表深拷贝。unordered_map<Node*, Node*> 存原节点→克隆节点的映射。DFS 递归遍历：已克隆则直接返回；否则创建新节点，先存入 map 再递归克隆邻居，最后返回克隆节点。先存再递归是关键，防止环导致死循环。

🔑 关键代码片段

📚 学到的知识点

  • 深拷贝 vs 浅拷贝：深拷贝所有节点都是 new 出来的，修改新图不影响原图
  • 哈希表记录映射：原节点 → 克隆节点，避免重复创建
  • 先存再递归：先 visited[node]=clone 再遍历邻居，防止环导致死循环
  • DFS vs BFS 都能做：DFS 递归简洁，BFS 队列无栈溢出风险

⚠️ 易错点 & 反思

  • ❌ 忘记 visited[node]=clone 放在遍历邻居之前 → 有环时会死循环
  • ❌ visited 不传引用 → 每次递归拷贝一份 map，永远找不到已克隆的节点
  • ❌ 直接 return node → 浅拷贝，新图仍指向原节点
  • 💡 图 DFS 和树 DFS 的区别：树没有环，不需要 visited；图有环，必须记录已访问节点

🔗 关联题目

  • [200] Number of Islands — 网格图的 DFS
  • [138] Copy List with Random Pointer — 链表的深拷贝，思路类似
  • [133] Clone Graph — 图的深拷贝，map + DFS/BFS

[Medium] Course Schedule (207)

🎯 核心思路

有向图检测环 + 三色 DFS。邻接表建图，vector<int> status 三态标记：0=未访问、1=正在当前路径、2=已安全完成。DFS 递归中遇到 status=1 → 有环；status=2 → 安全跳过。所有课程都能安全完成返回 true。

🔑 关键代码片段

📚 学到的知识点

  • 有向图表示：邻接表 graph[p[1]].push_back(p[0])，节省空间
  • 三色 DFS：0=未访问、1=访问中（再遇到=环）、2=已安全（跳过）
  • 区别于无向图：无向图两个 bool（visited+visiting）就够了；有向图需要三态区分”正在访问”和”已遍历完”
  • 入度法：统计每门课的入度（前置课数量），每次删掉入度为 0 的课并更新邻居入度。处理完所有课 → 无环。天然可输出课程顺序

⚠️ 易错点 & 反思

  • ❌ status[node]=1 时用 !status[neighbor] 跳过 → 错过环检测；必须显式判断 status[neighbor]==1
  • ❌ 提前标记 visited → 回溯时不会重新检查，漏判环
  • 💡 三色状态是图 DFS 检测环的标准模板，记住：1 是❌，2 是✅
