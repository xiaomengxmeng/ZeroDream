[Medium] Surrounded Regions (130)

🎯 核心思路

逆向思维 + 三步法。不从内部找被包围的 O，而是从四条边上的 O 出发 DFS，把所有与边界连通的 O 标记为临时字符 'T'（它们不会被包围）。然后遍历全盘：剩下的 'O' 是被包围的 → 变 'X'；'T' 恢复为 'O'。

🔑 关键代码片段

📚 学到的知识点

  • 反向思维：不找”被包围的”，而是找”不被包围的”——从边界出发逆向操作
  • 三步法模板：① 边界 DFS 标记 ② 全盘处理 ③ 恢复标记
  • 正向 vs 逆向：200 是正向找岛屿，130 是逆向排除边界连通区域
  • 原地标记：用临时字符 'T' 代替 visited 数组

⚠️ 易错点 & 反思

  • ❌ 直接遍历全盘改 'O'→'X' → 边界的 O 也被改了（必须先用 T 标记保护）
  • ❌ DFS 条件用 board[i][j] == 'O' 而不是 != 'O' → T 和 X 也会进 DFS
  • 💡 “逆向思维”是算法题中常用的技巧：正面难解决的问题，反过来想可能很简单

🔗 关联题目

  • [200] Number of Islands — 正向找连通分量
  • [695] Max Area of Island — 正向求面积
  • [130] Surrounded Regions — 逆向排除边界连通分量

[Medium] Pacific Atlantic Water Flow (417)

🎯 核心思路

双向 DFS + 取交集。倒过来想：不从高地往边界流，而是从两大洋边界（太平洋=上+左，大西洋=下+右）反向 DFS，从低往高走（heights[next] >= heights[cur]）。两个 visited 矩阵分别标记能流到太平洋/大西洋的格子。最后遍历全盘，取交集。

🔑 关键代码片段

📚 学到的知识点

  • 双向反向 DFS：从两大洋边界反向 DFS（从低往高走），等价于”哪些格子能流到该大洋”
  • 两个 visited 取交集：同时能流到两大洋的格子 = 答案
  • 130 vs 417 对比：都是”从边界 DFS 向内”的思路，130 是一个边界判包围，417 是两个边界取交集

⚠️ 易错点 & 反思

  • ❌ 边界 DFS 起点只从四个角开始 → 必须遍历整条左/上/右/下边界
  • ❌ Visited[i][j]=0 回溯 → 标记过的格子不能再清除，否则重复 DFS 死循环
  • ❌ 笔误 heights[i-1][j]>=heights[i-1][j] → 应该是 >= heights[i][j]
  • 💡 流水的正向思维 → 逆向 DFS：正向不好做，反向标记更简单

🔗 关联题目

  • [130] Surrounded Regions — 单边界逆向 DFS
  • [417] Pacific Atlantic Water Flow — 双向界逆向 DFS + 取交集
  • [200] Number of Islands — 正找连通分量

[Medium] Rotting Oranges (994)

🎯 核心思路

多源 BFS + 按层计时。所有初始腐烂橘子一起入队（多源），按层处理：每处理完一层，分钟数 +1。最后遍历全盘检查是否还有新鲜橘子。BFS 结束后，如果有新鲜橘子剩余 → -1，否则返回分钟数。

🔑 关键代码片段

📚 学到的知识点

  • 多源 BFS：多个起点同时入队，模拟”同步扩散”
  • 按层计时：用 size = q.size() 和 while (size--) 把 BFS 分层，每层 = 1 分钟
  • DFS vs BFS 选择：求最短时间/同步扩散 → BFS；求连通分量/全部遍历 → DFS
  • 事后检查：BFS 结束后遍历全盘，确认所有新鲜橘子都已腐烂

⚠️ 易错点 & 反思

  • ❌ count++ 同时出现在内层 pop 和外层 → 分钟数翻倍
  • ❌ if (i==m-1 && j==n-1) return count → 错误的结束条件，应等 BFS 完统一检查
  • ❌ 忘记最后遍历检查还有没有新鲜橘子 → 可能返回分钟数但实际有橘子烂不到
  • 💡 按层 BFS 模板：size = q.size() → while (size--) 处理当前层 → 处理完后分钟+1

  • [994] Rotting Oranges — 多源 BFS + 按层计时
