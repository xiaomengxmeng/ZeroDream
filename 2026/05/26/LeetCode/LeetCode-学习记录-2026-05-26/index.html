[Medium] Search in Rotated Sorted Array (33)

🤔 我的原始思路

[在此记录你的初步想法]

  • 🧩 旋转数组的特征 ：数组是 两段升序 拼接——左段大、右段小。给一个 mid，你怎么判断 mid 在左段还是在右段？
  • 没思绪
  • 🎯 判断 target 在哪 ：知道 mid 在左段后，怎么判断 target 是在 mid 左边还是右边？比如说 nums=[4,5,6,7,0,1,2] ， mid 指向 6（左段）， target=1 （在右段）——这时应该往哪边搜索？
  • 这时候看nums[0] 于target 比较  < 左边 > 右边
  • ⚖️ 均匀性 ：和标准二分的区别是什么？什么条件下 nums[mid] < target 不一定是往右搜索？
  • nums[0] <target

🔍 我的思考过程

  • 二分查找的变体，先判断 mid 在左段还是右段（和 nums[0] 比较）
  • 再判断 target 在哪一段，决定收缩方向
  • 左段条件：nums[0] <= target && target < nums[mid] 往左搜，否则往右
  • 右段条件：nums[0] > target && nums[mid] < target 往右搜，否则往左

💡 对比收获

  • 标准二分 vs 旋转二分：标准二分只要比大小；旋转二分先判断段，再判断方向
  • 关键条件：nums[0] <= nums[mid] 判断 mid 所在段，等号处理 mid=0
  • 边界易错：nums[0] <= target 包含等号，漏了就找不到 target=nums[0]

[Medium] Search in Rotated Sorted Array (33)

🎯 核心思路

旋转数组二分：先判断 mid 在左段还是右段，再判断 target 在哪段来决定收缩方向。O(log n)

🔑 关键代码片段

📚 学到的知识点

  1. 旋转数组分段：nums[0] <= nums[mid] 判断 mid 在左段，否则在右段
  2. 段内判断：左段内 nums[0] <= target < nums[mid] 才往左搜；右段内 nums[mid] < target < nums[0] 才往右搜
  3. 等号处理：nums[0] <= nums[mid] 和 nums[0] <= target 的等号不能少，否则漏掉边界

⚠️ 易错点 & 反思

  • ❌ nums[0] < nums[mid] 漏等号 → mid=0 时误入 else 分支
  • ❌ nums[0] < target 漏等号 → target=nums[0] 时找不到
  • ❌ 右段分支条件写反 → 复制左段条件没改
  • ❌ 旋转数组不能用标准二分直接比大小
