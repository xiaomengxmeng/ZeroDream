[Easy] Convert Sorted Array to Binary Search Tree (108)

🤔 我的原始思路

升序数组转换为一棵高度平衡二叉搜索树

二分法选到根节点再逐个构建子树

🔍 我的思考过程

  1. 升序数组 → 中间元素是 BST 的根节点
  2. 递归：左半区间构建左子树，右半区间构建右子树
  3. 递归终止条件：left > right（区间无效）
  4. mid 用 left + (right - left) / 2 防止溢出
  5. 分两个函数：主函数调用、辅助函数递归

💡 对比收获

代码与标准答案一致 ✅

  1. 有序数组 + BST + 高度平衡 → 二分递归是标准解法
  2. 与二分查找同源，思想相通：取中间、分两半、递归处理
  3. 防止溢出的写法 left + (right - left) / 2 要养成习惯
  4. 辅助递归函数的设计模式：传入区间边界，返回子树的根

[Medium] Kth Smallest Element in a BST (230)

🤔 我的原始思路

  • BST 的中序遍历 = 递增序列
  • 第 k 小元素 = 中序序列的第 k 个节点
  • 不需要遍历整棵树，找到第 k 个就能提前终止

🔍 我的思考过程

  1. 用迭代中序遍历模板（与94题相同）
  2. 左链入栈 → 出栈访问 → 切到右子树，按序收集节点值
  3. 全部节点值存入 vector<int> vals
  4. 最后返回 vals[k-1]
  5. 下一次可以用计数优化：count == k 时直接 return

💡 对比收获

代码正确 ✅

  1. 当前解法 O(n) 时间 + O(n) 空间
  2. 进阶优化：边遍历边计数，找到第 k 个直接返回，O(k) 时间 + O(1) 空间
  3. BST 解题核心：中序遍历递增 是 BST 题最重要的性质
  4. 迭代中序模板要背熟：外层 while + 内层 while 左链入栈

[Medium] Lowest Common Ancestor of a BST (235)

🤔 我的原始思路

LCA 最近公共祖先

BST 二叉搜索树

如果 p, q 分别在根节点的两侧，根就是 LCA

在同侧 < 就对左子树重复 > 就对右子树重复

🔍 我的思考过程

  1. BST 性质：左子树所有节点 < 根 < 右子树所有节点
  2. p 和 q 都在右侧 → LCA 在右子树，递归 root->right
  3. p 和 q 都在左侧 → LCA 在左子树，递归 root->left
  4. 一个在左一个在右 → 当前根就是 LCA
  5. 递归终止条件隐式包含：p 或 q 就是 LCA 的情况（此时二者分居两侧）

💡 对比收获

代码正确 ✅

  1. BST LCA 比普通 LCA 简单太多：普通二叉树需要后序遍历 + 回溯，BST 只需比较值
  2. 三路判断的思维：全右 → 递归右 / 全左 → 递归左 / 其他 → 返回当前
  3. 可以改迭代：while (true) { if (p,q都在右) root=root->right; else if(都在左) root=root->left; else return root; }
  4. 这题为 [236] LCA of Binary Tree 做铺垫

[Easy] Convert Sorted Array to Binary Search Tree (108)

🎯 核心思路

升序数组 → 高度平衡 BST = 二分递归法。每次取数组中间元素作为根节点，左半部分构建左子树，右半部分构建右子树。与二分查找同源。

🔑 关键代码片段

📚 学到的知识点

  • BST + 有序数组：有序数组的中间节点 = BST 的根节点
  • 递归构建模板：选根 → 左区间递归 → 右区间递归
  • **mid = left + (right - left) / 2**：防止 left+right 溢出
  • 递归终止条件：left > right 返回 nullptr

⚠️ 易错点 & 反思

  • ❌ 忘记递归终止条件 → 无限递归，栈溢出
  • ❌ mid 计算时直接用 (left+right)/2 → left+right 可能溢出
  • ✅ 用 left + (right - left) / 2 更加安全
  • 💡 核心模式：二分+分治，与二分查找是同源思想

[Medium] Kth Smallest Element in a BST (230)

🎯 核心思路

BST 中序遍历 = 递增序列，第 k 小 = 中序序列第 k 个元素。用迭代中序遍历依次取出节点值，取 vals[k-1]。

🔑 关键代码片段

📚 学到的知识点

  • BST 性质：中序遍历 = 升序序列，第 k 个即第 k 小
  • 迭代中序模板：左链入栈 → 出栈访问 → 切右子树
  • 进阶优化：不需要存完整数组，找到第 k 个直接返回，提前终止

⚠️ 易错点 & 反思

  • ✅ 中序记住：左-根-右，栈模拟的关键是”先一路向左走到底”
  • 💡 优化点：可以边遍历边计数，count == k 时直接 return，避免遍历整棵树
  • 💡 当前解法 O(n) 空间存数组，优化到 O(1) 空间

[Medium] Lowest Common Ancestor of a BST (235)

🎯 核心思路

利用 BST 性质：左<根<右。若 p、q 都在右侧 → 递归右子树；都在左侧 → 递归左子树；分居两侧 → 当前根即为 LCA。

🔑 关键代码片段

📚 学到的知识点

  • BST 的 LCA 比普通二叉树简单：普通二叉树需要后序遍历，BST 只需比较值
  • 判断标准：p 和 q 分居 root 两侧 → root 即为 LCA
  • 递归方向：都在右 → 往右走；都在左 → 往左走
  • 迭代版本：可将递归改为 while 循环，O(1) 空间

⚠️ 易错点 & 反思

  • ✅ 条件判断用 root->val < p->val && root->val < q->val，不是 p->val < root->val
  • 💡 迭代优化：可以改成 while 循环，不用递归栈，空间 O(1)
  • 💡 这题是 LCA 系列的核心基础：BST → 带父指针 → 普通二叉树

🔗 关联题目

  • [236] LCA of a Binary Tree - 普通二叉树版，需要后序遍历
  • [108] 有序数组转 BST - 也是利用 BST 有序性质
